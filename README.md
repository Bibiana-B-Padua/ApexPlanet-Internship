# ApexPlanet Data Analytics Internship - Task 1


## Objective
This repository contains the deliverables for **Task 1: Data Immersion & Wrangling** of the ApexPlanet Data Analytics Internship. The primary goal of this task is to acquire, profile, clean, and prepare the provided `Sales_Dataset` for downstream analysis.

## Files in this Repository
* **`README.md`**: Contains the project overview and Data Dictionary.
* **`data_wrangling.py`**: A Python/Pandas script that performs initial data profiling, handles duplicates, standardizes formats, and engineers new features.
* **`Cleaned_Sales_Dataset.csv`**: The final, analysis-ready dataset generated by the cleaning script.

---

## Data Dictionary

Below is the documentation for the variables present in the initial `Sales_Dataset`, including their types and business relevance.

| Column Name | Expected Data Type | Description | Potential Business Relevance |
| :--- | :--- | :--- | :--- |
| **Order_ID** | String | Unique alphanumeric identifier for each transaction (e.g., ORD100002). | Essential for tracking specific transactions, resolving disputes, and counting total order volume. |
| **Order_Date** | Date | The date the transaction occurred (YYYY-MM-DD). | Crucial for time-series analysis, identifying seasonal trends, and forecasting future sales. |
| **Customer_ID** | String | Unique identifier for the purchaser (e.g., CUST5529). | Allows for tracking repeat purchases, customer lifetime value (CLV) calculations, and cohort analysis. |
| **Customer_Name** | String | Anonymized name of the customer. | Mostly for reporting readability; usually secondary to `Customer_ID` for analysis. |
| **Age** | Integer | The age of the customer in years. | Useful for demographic segmentation and targeted marketing campaigns. |
| **Gender** | String | The gender of the customer (Male/Female). | Demographic profiling and understanding product preferences across different genders. |
| **City** | String | The city where the order was placed. | Geographic performance tracking, regional demand analysis, and localized marketing. |
| **Product** | String | The specific item purchased. | Inventory management, product popularity analysis, and cross-selling strategies. |
| **Category** | String | The broader classification of the product. | High-level sales performance tracking and category-based budget allocation. |
| **Quantity** | Integer | The number of units of the product purchased in the order. | Understanding purchase volume and inventory depletion rates. |
| **Unit_Price** | Float | The price of a single unit of the product. | Pricing strategy analysis and revenue modeling. |
| **Total_Sales** | Float | Total revenue for the order line (`Quantity * Unit_Price`). | The primary metric for revenue tracking, financial reporting, and profitability analysis. |

## Data Cleaning & Transformation Process
The `data_wrangling.py` script performs the following steps to ensure data quality:
1. **Profiling:** Checks for missing values and initial duplicate counts.
2. **Deduplication:** Drops exact duplicate rows to prevent skewed sales metrics.
3. **Standardization:** * Converts `Order_Date` to standard datetime objects.
    * Applies Title Case to categorical text fields (`Gender`, `City`, `Category`) to resolve inconsistencies (e.g., 'male' vs 'Male').
4. **Feature Engineering:** Creates a new `Age_Group` categorical column derived from the `Age` column to assist in demographic segmentation.
5. **Validation:** Verifies that `Total_Sales` accurately reflects `Quantity * Unit_Price`.
